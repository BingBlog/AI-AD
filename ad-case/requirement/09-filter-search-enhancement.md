# 筛选字段搜索功能需求

## 功能描述

在案例检索的筛选条件中，支持**品牌名称**、**品牌行业**、**活动类型**、**活动地点**四个字段的搜索功能。搜索的内容需要是当前数据库中实际存在的数据范围。

## 业务价值

1. **提升用户体验**：

   - 用户可以通过输入部分关键词快速筛选，无需输入完整匹配值
   - 减少因输入不准确导致的筛选失败
   - 帮助用户了解数据库中实际可用的筛选值

2. **提高筛选效率**：
   - 支持模糊匹配，提高筛选的容错性
   - 提供输入提示，帮助用户快速选择有效的筛选值
   - 统一四个筛选字段的搜索体验

## 功能需求

### 核心需求

1. **筛选字段模糊搜索**：

   - 品牌名称、品牌行业、活动类型、活动地点四个字段均支持模糊搜索
   - 用户输入部分关键词即可匹配到相关的筛选值
   - 例如：输入"汽车"可以匹配"汽车"、"汽车、家电"等包含"汽车"的值

2. **数据范围限制**：
   - 筛选功能应基于数据库中实际存在的数据
   - 确保用户能够了解可用的筛选值范围
   - 避免无效输入导致的筛选失败

### 增强需求（可选）

3. **输入提示功能**：

   - 在用户输入筛选值时，提供候选值提示
   - 候选值列表来自数据库中实际存在的数据
   - 显示每个候选值对应的案例数量

4. **自动补全功能**：
   - 支持输入时自动显示匹配的候选值
   - 用户可以通过键盘导航或点击选择候选值
   - 也可以直接输入任意文本进行搜索

## 功能详细说明

### 品牌名称搜索

- 支持输入品牌名称的部分关键词进行模糊搜索
- 例如：输入"耐克"可以匹配"耐克"、"耐克中国"等
- 支持显示匹配的品牌名称列表和对应的案例数量

### 品牌行业搜索

- 支持输入行业名称的部分关键词进行模糊搜索
- 例如：输入"汽车"可以匹配"汽车"、"汽车、家电"等
- 支持显示匹配的行业名称列表和对应的案例数量

### 活动类型搜索

- 支持输入活动类型的部分关键词进行模糊搜索
- 例如：输入"影视"可以匹配"影视"、"影视广告"等
- 支持显示匹配的活动类型列表和对应的案例数量

### 活动地点搜索

- 支持输入地点名称的部分关键词进行模糊搜索
- 例如：输入"中国"可以匹配"中国大陆"、"中国香港"等
- 支持显示匹配的地点名称列表和对应的案例数量

## 用户体验要求

### 输入体验

1. **输入方式**：

   - 支持文本输入框
   - 支持直接输入或从候选列表中选择

2. **输入提示**：

   - 输入时实时显示匹配的候选值（如有）
   - 显示每个候选值对应的案例数量
   - 提示信息清晰，不影响用户正常输入

3. **输入容错**：
   - 支持部分关键词匹配
   - 支持大小写不敏感
   - 输入空白字符应自动去除

### 交互体验

1. **筛选响应**：

   - 筛选条件变化时，实时更新检索结果
   - 筛选结果为空时，给出明确提示

2. **筛选条件展示**：
   - 清晰展示当前生效的筛选条件
   - 支持快速清除单个或所有筛选条件

   - 用户输入部分关键词即可进行筛选

### P1 - 重要功能（应该实现）

2. **输入提示功能**：

   - 提供候选值查询功能
   - 在用户输入时显示匹配的候选值列表

3. **自动补全功能**：
   - 支持输入时自动显示匹配选项

## 非功能需求

### 性能要求

- 筛选响应时间应在 1 秒内
- 候选值查询响应时间应在 500 毫秒内
- 支持大数据量下的模糊搜索性能

### 可用性要求

- 筛选功能应稳定可靠，不影响其他筛选条件
- 候选值列表应准确反映数据库中的实际数据
- 支持筛选条件的保存和复用（如已有功能）

## 验收标准

### 功能验收

1. 四个筛选字段均支持模糊搜索
2. 输入部分关键词能够匹配到相关结果
3. 候选值列表（如实现）显示的值为数据库中实际存在的值
4. 筛选功能不影响其他筛选条件和检索功能

### 用户体验验收

1. 用户可以通过输入部分关键词快速筛选
2. 输入提示（如实现）清晰有用，不影响正常输入
3. 筛选结果符合用户期望
4. 筛选失败时有明确提示

## 相关功能

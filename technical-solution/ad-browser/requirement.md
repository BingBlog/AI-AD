# 一、项目背景与目标

## 1.1 背景

广告营销策划师在做方案、比稿、创意时，核心痛点是：

- 需要大量参考 **小红书 / 抖音 / 公开平台** 的真实营销案例
- 搜索成本高、整理耗时、经验难以沉淀
- 案例“看过就忘”，无法形成系统知识

与此同时：

- 直接做爬虫或内容聚合，**法律与平台风险极高**
- AI Agent + Browser Automation 技术逐渐成熟

## 1.2 产品目标

打造一个：

> **面向广告营销行业的「研究型智能平台」**

核心目标：

- 帮助策划师 **高效研究公开营销案例**
- 将“碎片内容”转化为 **结构化、可复用的营销知识**
- 沉淀行业级 **营销洞察知识库**

# 二、核心产品定位（非常重要）

## 2.1 正确定位（必须坚持）

> **不是内容平台** > **不是案例搬运站** > **而是：营销研究与洞察平台**

平台只做：

- 研究
- 抽象
- 总结

不做：

- 原文分发
- 原视频存储
- 替代原平台阅读

## 2.2 用户画像

- 广告公司策划 / 创意 / 策略
- 品牌市场部
- 内容营销顾问

核心需求：

- 快速找到「类似案例」
- 总结「可复用的方法论」
- 为方案、提案提供依据

# 三、整体产品架构总览

```
浏览器插件（前端入口）
    │
    ▼
本地 Agent（用户设备）
    │  浏览 / 操作 / 提取
    ▼
结构化摘要 & 洞察（本地生成）
    │
    │ 用户确认 / 编辑
    ▼
平台云端知识库（抽象知识）
    │
    ▼
搜索 / 复用 / 洞察生成
```

**核心原则：**

- 原始内容 ≈ 停留在用户本地
- 平台只存 **不可还原原文的知识**

# 四、Browser Use & 本地 Agent 方案

## 4.1 为什么使用本地 Agent + Browser Use

原因：

- 使用 **用户自己的浏览器环境**（账号、Cookie、风控最低）
- 节省云端算力与带宽
- 避免集中式爬虫风险

定位：

> 本地 Agent = 用户的「研究助理」

## 4.2 Browser Use 的角色

Browser Use 负责：

- 搜索关键词
- 翻页
- 判断是否点进详情
- 页面基础理解

不负责：

- 长文本总结
- 创意输出

## 4.3 Browser Use 推荐模型

### 执行模型（默认）

- GPT-4.1-mini（首选）
- Claude 3.5 Sonnet（更稳）
- DeepSeek Chat（性价比）

### 原则

- 指令遵循 > 推理深度
- 稳定 > 聪明

# 五、Agent Prompt 设计（核心能力）

## 5.1 Prompt 总体拆分

### 1️⃣ 搜索子 Prompt

- 明确关键词
- 指定平台
- 约束搜索范围

### 2️⃣ 筛选子 Prompt

- 判断是否与主题强相关
- 是否值得进入详情

### 3️⃣ 摘要子 Prompt

- 提炼营销要素
- 不复制原文

### 4️⃣ 洞察子 Prompt

- 抽象方法
- 提炼可复用经验

## 5.2 Prompt 核心原则

- 明确「禁止行为」（不登录、不点赞、不评论）
- 限制步骤数量
- 强制结构化输出

# 六、自动执行与用户协作机制

## 6.1 为什么不能全自动

- 平台风控
- 合规风险
- 内容判断主观性

## 6.2 人机协作模式

```
Agent 自动执行
   ↓
关键节点停下
   ↓
用户确认 / 修正
   ↓
继续执行
```

示例节点：

- 是否相关
- 是否加入知识库

# 七、性能与效率优化方案

## 7.1 自动执行慢的原因

- 多轮 LLM 决策
- 页面加载
- DOM 分析

## 7.2 优化策略

### 技术层

- 合并步骤
- 减少截图
- 限制最大操作数

### 模型层

- 执行模型与总结模型分离
- 优先使用小模型决策

### 产品层

- 并行跑多个标签页
- 用户可随时中断

# 八、知识库设计（平台核心资产）

## 8.1 知识库不存什么

- 原文全文
- 原视频
- 原图片

## 8.2 知识库 Schema（示例）

- 行业：汽车
- 节点：春节营销
- 品牌：比亚迪
- 平台：小红书
- 创意类型：情感叙事
- 传播机制：KOL 联动
- 核心洞察：

  - 节日情绪绑定品牌价值
  - 用户故事降低广告感

- 来源链接（跳转）

# 九、合规与边界设计（生死线）

## 9.1 明确合规边界

允许：

- 研究
- 摘要
- 引用链接

禁止：

- 批量无人采集
- 原文再分发
- 自动同步用户浏览内容

## 9.2 对外官方表述（非常重要）

> 本平台是广告营销行业的 **研究与洞察工具**，
> 用于辅助用户分析公开信息，
> 不存储、不分发第三方平台原始内容。

# 十、LLM API Key 管理方案

## 10.1 核心原则

> 本地 Agent 永远不持有主模型 Key

## 10.2 推荐方案

### 默认：云端 LLM Proxy

```
本地 Agent → 你 API → 大模型
```

- Key 只存在云端
- 可控成本
- 可停用用户

### 可选：BYOK

- 高级用户使用自己的 Key

## 10.3 Token 签发

- 短期 Token（10–30 分钟）
- 可撤销
- 可限功能

# 十一、客户端形态与安装

## 11.1 是否需要安装软件

需要：

- 本地 Agent（macOS / Windows）
- 浏览器插件（入口 & UI）

## 11.2 插件如何引导安装 Agent

- 首次使用检测本地 Agent
- 未安装时：

  - 提示 + 一键下载
  - 安装指引

## 11.3 本地 Agent 打包

- 技术：Python + PyInstaller
- macOS：打包为 .dmg
- 自动更新机制

# 十二、典型用户流程示例

1. 用户输入：

   > 春节 · 汽车营销 · 小红书

2. 本地 Agent 搜索 + 筛选
3. 自动提取 10 条案例摘要
4. 用户勾选有价值案例
5. 平台生成：

   - 方法论总结
   - 可复用洞察

# 十三、成本与规模预期

- 单案例执行：约 20–40 秒
- 10 条案例：约 5–8 分钟
- 成本集中在：

  - 执行模型调用
  - 总结模型调用
